App_Features

here we describe some key features ( add document , add request , authentification (signup , login , logout)

add Document Request

Seeker user has the ability to create request to report the loss of his document
Finder User can also create request to declare that the document has been found

View Document Request

    User can :
  • view the status of request (exemple Awaiting_reply_owner: wainting other user to declare the found of document)
  • delete: destroy the created request
  • info : show the description of created request
  • contact: show a button to display the contact information of other user that finds the document
  • doc: display the description of document

add Document

Seeker User can create Document to add more description
so the Finder User can view this information and confirm the claim of Seeker

    also user can use the View Documents Button to :
  • view created document
  • delete : destroy document
  • info : view or update document

Authentification

user can click button sign up to create new account
also he can logged in using the button Login to access the application
when the use is logged in :
in the top right corner there is a button to help the user logged out from the application

Since childhood, a friend and I would guard our sheep in the village.
Each of us possessed a stone, which we would hurl into the tall grass.
The challenge was to locate our own stone before the other did,
turning it into a game of skill and observation.
This experience instilled in me a love for uncovering hidden things.
It's why I find myself drawn to projects like this one,
where the task is to find vital missing documents,
applying that same sense of determination and attention to detail to real-world applications
